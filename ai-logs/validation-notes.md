# Validation Notes

Date: 2026-04-28

## How We Checked AI-Assisted Output

1. Coursework checklist review
   - The team compared the webpage sections against the project requirements: Motivation and Research, User Requirements, Ideation and Alternatives, Technical Implementation, Evaluation and Reflection, and AI disclosure.

2. Requirement alignment review
   - R1, R2, and R3 were checked against design goals, prototype features, research evidence, and Alpha testing evidence.
   - The team corrected the R3 row so location sharing was described as an iteration added after Alpha testing, not as an original Alpha feature.

3. User-testing evidence review
   - The team replaced generic AI-written testing descriptions with the actual User A, User B, and User C findings.
   - User identities were anonymized.
   - Testing data was checked for consistency with the Alpha testing section.

4. Screenshot and asset review
   - The before-and-after screenshots were added from `images/iteration`.
   - File names were changed to stable English names.
   - Image paths were checked locally before committing.

5. Ethical and accessibility review
   - The team checked that user consent and anonymity were mentioned.
   - Image alt text was added for portfolio screenshots.
   - AI-generated text was reviewed to reduce generic claims, unsupported user data, and misleading statements.

## What Worked

- AI was useful for organizing section structure, improving English wording, and making HTML/CSS changes faster.
- AI helped turn teacher feedback into concrete portfolio sections.

## What Did Not Work

- Some AI drafts were too generic and needed to be replaced with real project evidence.
- Some wording implied features existed earlier than they actually did, so the team corrected the timeline.
- The team still needed to manually check layout, image paths, and whether the content matched the real Alpha test.
